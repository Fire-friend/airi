> **Migration note**: Screen observation was previously powered by screenpipe. Screenpipe has been removed. MineContext is now the only supported backend. If you had screenpipe configured, uninstall it and follow this guide to set up MineContext instead.

## What This Does

When MineContext is running alongside AIRI, the desktop app gains two observation tracks:

- **Near-realtime track** — individual screenshots captured every few seconds, giving AIRI a live sense of which app and window you are in right now. Requires screenshot capture to be enabled (see below).
- **Long-memory track** — AI-synthesized activity summaries generated by MineContext roughly every 10 minutes, building a longer-term picture of your usage patterns over time.

Together, these let AIRI understand your device and usage habits automatically rather than asking you to describe them.

## 1. Install and Start MineContext

MineContext is distributed as a Python package called `opencontext`.

```bash
pip install opencontext
opencontext start --host localhost --port 1733
```

Leave MineContext running in the background. AIRI connects to it at `http://127.0.0.1:1733`.

## 2. Enable Screenshot Capture (Required for Near-Realtime Track)

Screenshot capture is **off by default** in MineContext. Without it, only the long-memory track produces data — the near-realtime "what app am I in right now" track stays empty.

To enable it:

1. Open `~/.opencontext/config/config.yaml` in a text editor.
2. Add or set the `capture_interval` field (value is seconds; 5 is a reasonable starting point):

   ```yaml
   capture_interval: 5
   ```

3. Restart MineContext (`Ctrl-C` then `opencontext start` again).
4. In AIRI Settings → **Screen Observation** → **MineContext Connection**, toggle **Screenshot Capture** on.

Both steps are required: the config tells MineContext to take screenshots, and the toggle tells AIRI to read them.

## 3. Enable Observation in AIRI Settings

1. Open AIRI Settings → **Screen Observation**.
2. Flip the **Enable screen observation** switch on.
3. Add at least one app to the **Allowed apps** list. Only apps on this list have their context forwarded to the persona. An empty list means observation is on but nothing is watched — the status bar shows "Not observing (no apps selected)" as a reminder.

The connection indicator (green dot = connected, grey = not connected) shows whether MineContext is reachable. If it stays grey, check that `opencontext start` is still running.

## Configuration Reference

All settings live in Settings → **Screen Observation** → **MineContext Connection**.

| Setting | Default | Minimum | Description |
|---|---|---|---|
| Service address | `http://127.0.0.1:1733` | — | Change if MineContext runs on a non-default host or port. Takes effect immediately. |
| Screenshot capture | Off | — | Must also be enabled in MineContext config (see above). |
| Current-state poll interval | 15 000 ms | 5 000 ms | How often AIRI reads near-realtime screenshot data. |
| Long-memory poll interval | 30 000 ms | 10 000 ms | How often AIRI reads synthesized activity summaries. |

Interval changes take effect immediately without restarting.

## Privacy Controls

### App Whitelist

Only apps you name in **Allowed apps** can contribute context to the persona. Apps not on the list are silently ignored — they never reach AIRI even if MineContext has captured them.

App names are matched case-insensitively. Use the name as it appears in your operating system's app list (for example `Code`, `Slack`, `Safari`).

### Pause Observation

Press **Ctrl+Alt+P** (or **Cmd+Alt+P** on macOS) at any time to pause observation for one hour. From the same shortcut, press again to resume. You can also set the duration explicitly via the pause menu in Settings.

Pause options available in the menu:
- Pause for 15 minutes
- Pause for 1 hour
- Pause for the rest of today

### Automatic Suppression

AIRI automatically suppresses observation (stops sending context to the persona) when:

- A fullscreen app is detected — privacy by default in focused immersive use.
- A meeting application is in the foreground — names, screen content, and meeting participants are never observed.

Observation resumes automatically when you leave the suppressed surface.

### What Goes Into the Persona

MineContext activity summaries pass through the allowedApps filter before they reach the persona. Only the content of whitelisted apps is included. Summaries that contain no whitelisted-app content are discarded.

Synthesized summaries only enter the persona's long-term understanding once they reach sufficient confidence and importance thresholds — low-quality or ambiguous captures are dropped rather than accumulated as noise.

## Troubleshooting

**Connection indicator stays grey**
- Check that `opencontext start` is running and has not crashed.
- Verify the service address in settings matches the port you started MineContext on.

**Near-realtime track shows no data (current-state always empty)**
- Confirm `capture_interval` is set in `~/.opencontext/config/config.yaml`.
- Confirm you restarted MineContext after editing the config.
- Confirm the Screenshot Capture toggle is on in Settings.

**Observation status shows "Not observing (no apps selected)"**
- Add at least one app name to the Allowed apps list in Settings.
