> **迁移说明**：屏幕观察功能原先由 screenpipe 驱动。screenpipe 已从项目中移除，MineContext 是目前唯一支持的后端。如果你此前配置过 screenpipe，请卸载并按本文重新配置 MineContext。

## 这个功能做什么

MineContext 在 AIRI 旁运行时，桌面版会开启两条观察轨道：

- **近实时轨** — 每隔几秒捕获一张截图，让 AIRI 实时感知你当前所在的应用和窗口。需要显式开启截图采集（见下文）。
- **长记忆轨** — MineContext 大约每 10 分钟生成一次由 AI 合成的活动摘要，随时间积累对你使用习惯的更长时间线理解。

两条轨道共同工作，让 AIRI 自动了解你的设备和使用模式，而不是靠你主动描述。

## 1. 安装并启动 MineContext

MineContext 以 Python 包 `opencontext` 的形式分发。

```bash
pip install opencontext
opencontext start --host localhost --port 1733
```

保持 MineContext 在后台运行。AIRI 会连接到 `http://127.0.0.1:1733`。

## 2. 开启截图采集（近实时轨必需）

MineContext 的截图采集**默认关闭**。不开启的话，只有长记忆轨有数据——"当前在哪个应用"的近实时轨会始终为空。

开启步骤：

1. 用文本编辑器打开 `~/.opencontext/config/config.yaml`。
2. 添加或设置 `capture_interval` 字段（单位为秒，5 是合理的起点）：

   ```yaml
   capture_interval: 5
   ```

3. 重启 MineContext（Ctrl-C 后再运行 `opencontext start`）。
4. 在 AIRI 设置 → **屏幕观察** → **MineContext 连接** 里，打开**截图采集**开关。

两步缺一不可：config 告诉 MineContext 拍截图，开关告诉 AIRI 去读取它们。

## 3. 在 AIRI 设置里启用观察

1. 打开 AIRI 设置 → **屏幕观察**。
2. 打开**启用屏幕观察**开关。
3. 在**允许的应用**列表里至少添加一个应用名。只有列表中的应用的上下文才会被传给人格。列表为空意味着观察已开启但什么都不采集——状态栏会显示"未观察（未选择任何应用）"提示你。

连接指示灯（绿点 = 已连接，灰点 = 未连接）显示 MineContext 是否可访问。如果一直是灰点，检查 `opencontext start` 是否还在运行。

## 配置项参考

所有设置位于 AIRI 设置 → **屏幕观察** → **MineContext 连接** 区块。

| 设置项 | 默认值 | 最小值 | 说明 |
|---|---|---|---|
| 服务地址 | `http://127.0.0.1:1733` | — | 如果 MineContext 运行在非默认主机或端口，在这里修改。改后立即生效，无需重启。 |
| 截图采集 | 关 | — | 还需要同步在 MineContext config 里配置（见上文）。 |
| 截图轮询间隔 | 15 000 ms | 5 000 ms | AIRI 读取近实时截图数据的频率。 |
| 活动摘要轮询间隔 | 30 000 ms | 10 000 ms | AIRI 读取合成活动摘要的频率。 |

间隔修改后立即生效，无需重启。

## 隐私控制

### 应用白名单

只有你填入**允许的应用**的应用才能向人格提供上下文。不在列表里的应用会被静默忽略——即使 MineContext 已采集，也不会传到 AIRI。

应用名不区分大小写。使用操作系统里显示的名称（例如 `Code`、`Slack`、`Safari`）。

### 手动暂停观察

随时按 **Ctrl+Alt+P**（macOS 为 **Cmd+Alt+P**）可暂停观察一小时，再按一次恢复。也可以在设置的暂停菜单里指定暂停时长：

- 暂停 15 分钟
- 暂停 1 小时
- 暂停至今天结束

### 自动抑制

以下情况 AIRI 会自动暂停向人格发送上下文：

- **全屏应用** — 沉浸式使用时默认保护隐私。
- **会议应用** — 会议期间姓名、屏幕内容、与会者等信息不会被观察。

离开上述状态后观察自动恢复。

### 什么内容会进入人格

MineContext 的活动摘要在进入人格之前会经过应用白名单过滤，只有白名单应用产生的内容才会被纳入。不含任何白名单应用内容的摘要会被丢弃。

合成摘要只有在置信度和重要性达到足够阈值后才会进入人格的长期理解——低质量或模糊的采集会被过滤掉，不会累积为噪声。

## 排查问题

**连接指示灯一直是灰点**
- 确认 `opencontext start` 还在运行，没有崩溃。
- 检查设置里的服务地址与实际启动端口一致。

**近实时轨没有数据（当前状态始终为空）**
- 确认 `~/.opencontext/config/config.yaml` 中已设置 `capture_interval`。
- 确认修改 config 后已重启 MineContext。
- 确认设置里的**截图采集**开关已打开。

**状态显示"未观察（未选择任何应用）"**
- 在设置的**允许的应用**列表中至少添加一个应用名。
